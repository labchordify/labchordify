How to start Contributing <b> </b> 
<p>Steps to Fork, Clone, and Create a Pull Request on GitHub:</p>

<ul>
  <li>Forking the Repository
    <ul>
      <li>Go to the original repository on GitHub.</li>
      <li>Click the "Fork" button (top-right corner).</li> 
    </ul>
  </li>

  <li>Cloning Your Fork
    <ul>
      <li>Access your forked repository on GitHub.</li>
      <li>Click the "Code" button and copy the provided URL.</li>
      <li>Open your terminal/command prompt and change to your desired workspace.</li> 
      <li>Clone the repository: `git clone <copied-URL>`</li>
    </ul>
  </li>

  <li>Working on Changes
    <ul>
      <li>Create a new branch: `git checkout -b <descriptive-branch-name>`</li>
      <li>Make changes to files using your preferred editor.</li>
      <li>Stage changes: `git add .` (or add specific files)</li>
      <li>Commit changes: `git commit -m "Clear description of your changes"`</li>
    </ul>
  </li>

  <li>Optional: Syncing with Original Repository
    <ul>
      <li>Add 'upstream' remote: `git remote add upstream <original-repo-URL>`</li>
      <li>Fetch changes: `git fetch upstream`</li>
      <li>Merge updates: `git merge upstream/master` (adjust branch name if necessary)</li>
    </ul>
  </li>

  <li>Pushing to Your Fork
     <ul>
        <li>Push your changes: `git push origin <descriptive-branch-name>`</li>
     </ul>
  </li>

  <li>Creating a Pull Request 
    <ul>
      <li>Go to your forked repository on GitHub.</li>
      <li>Click the "Compare & pull request" button or go to the "Pull requests" tab.</li>
      <li>Add a title and description for your pull request.</li>
      <li>Click "Create pull request". </li>
   </ul>
 </li>
</ul>

<h3> Important Notes: </h3>
<p> 

Consider communicating with the maintainers of the original repository before starting major changes to discuss whether your work aligns with the project's direction.
Keep your fork in sync with the original repository to avoid excessive merge conflicts.</p>
